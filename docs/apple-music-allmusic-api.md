# Apple Music API extraction notes for AllMusic

This note summarizes how `coloryr/AllMusic` loads music today and what can be safely reused from the public Cider source for an Apple Music provider.

## AllMusic loading model

AllMusic separates server-side music lookup from client-side playback:

1. The server registers providers in `AllMusic.MUSIC_APIS` by `IMusicApi#getId()`.
2. Search and queue commands store a `SongInfoObj` that carries an `id`, provider `api`, metadata, duration, and optionally a direct `playerUrl`.
3. When playback starts, the runtime resolves the stream URL with `api.getPlayUrl(songId)` unless `SongInfoObj#getPlayerUrl()` already contains a direct URL.
4. The server sends a `CommandType.PLAY` packet containing the URL to clients.
5. The client receives the URL and calls `AllMusicPlayer#setMusic(url)`, so a provider must ultimately supply a URL that AllMusic's Java player can open and decode.

The provider contract AllMusic needs is therefore small: `search`, `getMusic`, `getLyric`, `getPlayUrl`, playlist/list handling, ID parsing, and a busy-state guard.

## What Cider provides

The public Cider source is a MusicKit JS application. Its reusable parts are the Apple Music catalog and library API call patterns:

- Catalog lookup: `/v1/catalog/{storefront}/songs/{id}` with relationship includes such as artists and albums.
- Search: MusicKit/Apple Music catalog search by term and resource type, e.g. songs, albums, artists, playlists.
- Playback control: `MusicKit.getInstance().setQueue(...)`, `play()`, `pause()`, `skipToNextItem()`, and event listeners for now-playing metadata.
- Metadata extraction: normalize `nowPlayingItem.attributes`, `playParams`, artwork URL templates, duration, album, artist, and web URL.

Cider does **not** expose a plain, reusable audio file URL for Apple Music subscription tracks. MusicKit playback is handled inside the web runtime and Apple Music streams are DRM-protected. That means a normal AllMusic server-side `IMusicApi#getPlayUrl()` implementation cannot lawfully or reliably return a direct MP3/M4A/OGG URL for most Apple Music catalog songs.

## Safe integration options

### Option A: catalog/search-only provider

Implement an `IMusicApi` provider that supports search, metadata, lyrics metadata where available, and links, but returns `null` from `getPlayUrl()` for protected subscription tracks. This is compatible with the existing AllMusic model but cannot play protected Apple Music songs in the current Java audio pipeline.

### Option B: client-side MusicKit bridge

Add a new AllMusic playback path for Apple Music IDs where the client opens a MusicKit JS WebView or embedded browser and lets MusicKit perform playback after user authorization. In this design:

- The server sends an Apple Music play command containing `{storefront, type, id}` instead of a direct media URL.
- The client initializes MusicKit with a developer token generated by the project owner.
- The client calls `setQueue({ song: id })` or equivalent and then `play()`.
- Now-playing metadata and timing events are mirrored back into the AllMusic HUD.

This is the only practical way to reuse Cider's playback approach without attempting to extract DRM-protected stream URLs.

## Proposed provider API for AllMusic

```java
public interface AppleMusicApi extends IMusicApi {
    String getStorefront();

    AppleMusicCatalogSong getCatalogSong(String id);

    AppleMusicSearchPage searchCatalog(String term, int limit, int offset);

    AppleMusicPlaybackRef getPlaybackRef(String id);
}
```

`AppleMusicPlaybackRef` should be a data object for client-side MusicKit playback, not a direct audio URL:

```java
public record AppleMusicPlaybackRef(
        String storefront,
        String type,
        String id,
        String url
) {}
```

For compatibility with existing `IMusicApi`, the provider can map:

| AllMusic method | Apple Music behavior |
| --- | --- |
| `getId()` | Return `applemusic`. |
| `search(args, isDefault)` | Call `/v1/catalog/{storefront}/search?term=...&types=songs`. |
| `getMusic(id, player, isList)` | Call `/v1/catalog/{storefront}/songs/{id}` and map attributes to `SongInfoObj`. |
| `getLyric(id)` | Return empty lyrics unless the project has an authorized, documented source. |
| `getPlayUrl(id)` | Return `null` for protected catalog tracks; use `getPlaybackRef(id)` with a client MusicKit bridge instead. |
| `getMusicId(arg)` | Parse numeric Apple Music IDs and `/song/.../{id}` URLs. |
| `checkId(id)` | Accept catalog song IDs that are numeric strings. |

## Apple Music HTTP calls to implement

Use an Apple developer token generated by the AllMusic maintainer, plus a user music token for library/personalized endpoints.

```http
GET https://api.music.apple.com/v1/catalog/{storefront}/search?term={query}&types=songs&limit=25
Authorization: Bearer {developerToken}
```

```http
GET https://api.music.apple.com/v1/catalog/{storefront}/songs/{id}?include=artists,albums
Authorization: Bearer {developerToken}
```

```http
GET https://api.music.apple.com/v1/catalog/{storefront}/albums/{id}/tracks
Authorization: Bearer {developerToken}
```

Expected song fields for `SongInfoObj` mapping:

| Apple Music field | AllMusic field |
| --- | --- |
| `id` | `SongInfoObj.id` |
| `attributes.name` | name |
| `attributes.artistName` | author |
| `attributes.albumName` | album |
| `attributes.durationInMillis / 1000` | length seconds |
| `attributes.artwork.url` | image URL after replacing `{w}` and `{h}` |
| `attributes.url` | external link / playback reference URL |

## Cider-derived MusicKit client bridge shape

```js
async function playAppleMusicSong(id) {
  const mk = MusicKit.getInstance();

  if (!mk.isAuthorized) {
    await mk.authorize();
  }

  await mk.setQueue({ song: id });
  await mk.play();
}

function bindAppleMusicHudEvents(sendHudUpdate) {
  const mk = MusicKit.getInstance();

  mk.addEventListener(MusicKit.Events.nowPlayingItemDidChange, () => {
    const item = mk.nowPlayingItem;
    sendHudUpdate({
      id: item?.id ?? item?.playParams?.catalogId,
      name: item?.attributes?.name,
      artist: item?.attributes?.artistName,
      album: item?.attributes?.albumName,
      duration: item?.attributes?.durationInMillis,
      artwork: item?.attributes?.artwork?.url,
    });
  });

  mk.addEventListener(MusicKit.Events.playbackTimeDidChange, () => {
    sendHudUpdate({ currentTime: mk.currentPlaybackTime });
  });
}
```

## Implementation checklist

- Add an `applemusic` server provider for search and metadata.
- Keep direct URL playback disabled for protected Apple Music tracks.
- Add a new packet type for client-side provider playback, e.g. `APPLE_MUSIC_PLAY` with `AppleMusicPlaybackRef` JSON.
- Add an optional client WebView/MusicKit module to authorize and play Apple Music directly.
- Mirror MusicKit now-playing and time events into the existing AllMusic HUD packets.
- Document that users need an Apple Music subscription and an Apple Music developer token configured by the server/client owner.

## Starter code included in this repository

The repository includes `AppleMusicApi`, a Java `IMusicApi` implementation that performs the catalog/search portions above. It resolves song metadata through Apple Music's catalog API, maps results into AllMusic's `SongInfoObj` and `SearchPageObj`, parses plain IDs and Apple Music song URLs, and exposes `getPlaybackRef` for the MusicKit client bridge. Direct audio playback remains intentionally disabled through `getPlayUrl()` returning `null`.
